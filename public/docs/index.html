Introduction

This documentation aims to provide all the information you need to work with our API.

As you scroll, you'll see code examples for working with the API in different programming languages in the dark area to the right (or as part of the content on mobile). You can switch the language used with the tabs at the top right (or from the nav menu at the top left on mobile).

Base URL

http://localhost/authenticate/

Authenticating requests

To authenticate requests, include an Authorization header with the value "Bearer {TOKEN}".

All authenticated endpoints are marked with a requires authentication badge in the documentation below.

Als u zich aanmeld krijgt u een Json Web Token. als u account heeft neem dan contact op met ons

Authentication

APIs for authenticateion

Basic login

This is used for first time authentication. when your user has the role Deivce the devoceToken parameter is required

Example request:
curl --request POST \
    "http://localhost/authenticate/" \
    --header "Content-Type: application/json" \
    --header "Accept: application/json" \
    --data "{
    \"email\": \"development@dxmusic.nl\",
    \"password\": \"voluptatem\",
    \"deviceToken\": \"aliquid\"
}"

$client = new \GuzzleHttp\Client();
$response = $client->post(
    'http://localhost/authenticate/',
    [
        'headers' => [
            'Content-Type' => 'application/json',
            'Accept' => 'application/json',
        ],
        'json' => [
            'email' => 'development@dxmusic.nl',
            'password' => 'voluptatem',
            'deviceToken' => 'aliquid',
        ],
    ]
);
$body = $response->getBody();
print_r(json_decode((string) $body));
const url = new URL(
    "http://localhost/authenticate/"
);

const headers = {
    "Content-Type": "application/json",
    "Accept": "application/json",
};

let body = {
    "email": "development@dxmusic.nl",
    "password": "voluptatem",
    "deviceToken": "aliquid"
};

fetch(url, {
    method: "POST",
    headers,
    body: JSON.stringify(body),
}).then(response => response.json());

Renew token

requires authentication

With this endpoint you can request an new token, the current one should be valid.

Example request:
curl --request GET \
    --get "http://localhost/authenticate/jwt" \
    --header "Authorization: Bearer {TOKEN}" \
    --header "Content-Type: application/json" \
    --header "Accept: application/json"
$client = new \GuzzleHttp\Client();
$response = $client->get(
    'http://localhost/authenticate/jwt',

        'headers' => [

            'Content-Type' => 'application/json',
